# Failure Case Documentation Template

Failure is expected.
Denial is unacceptable.

This document defines how failures must be recorded.

---

## 1. Description

- What happened?
- When did it occur?
- Who was affected?

Avoid defensive language.
State facts plainly.

---

## 2. Ethical dimension

- Which values were involved?
- Which value conflicts were present?
- Was there unresolved moral tension?

---

## 3. System behavior

- What did the system do?
- Did it escalate?
- Did it suppress reflection?

---

## 4. Human response

- Who intervened?
- What decision was made?
- Was override justified?

---

## 5. Structural analysis

- Was this a design flaw?
- A governance flaw?
- A human misjudgment?
- External pressure?

Multiple causes may coexist.

---

## 6. Lessons

- What must change?
- What must not change?
- What remains uncertain?

---

## 7. Public disclosure status

- Disclosed
- Partially disclosed (with reason)
- Pending disclosure (with deadline)

Failure must not be erased.
It may be archived, but never rewritten.
